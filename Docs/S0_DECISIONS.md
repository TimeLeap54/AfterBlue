# S0 Locked Decisions

## D-S0-001
Week 04 implementation is the AfterBlue S0 feature baseline.

## D-S0-002
Current fishing covers cast, timed bite, bite-window input, success/failure result, collection registration, and persistence.

## D-S0-003
Reeling, Perfect Hook, fish behavior differences, stronger feedback, and balance changes are deferred to S1.

## D-S0-004
Movement during fishing remains allowed for S0. The acceptance condition is that movement must not duplicate bobbers, duplicate lines, duplicate rewards, or corrupt state.

## D-S0-005
Debug and temporary setup code can remain for S0 only if it is documented and does not create runtime errors in normal play.

## D-S0-006
S1 changes must be compared against this S0 documentation, build output, and manual test matrix.
